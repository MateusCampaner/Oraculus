Oraculus - Ajuda

1 - Como funciona a Configuração do modelo?

Para realizar as recomendações de colheita no sistema, o usuário deve selecionar um modelo, sendo ele o Modelo padrão do sistema ou algum modelo previamente treinado. Para se configurar um modelo, o usuário deve clicar no botão de Configurar Modelo, localizado na tela de Inserir Análise. Após isso o usuario poderá configurar as seguintes informações:

  • Proporção Teste / Treino
  • Número de vizinhos
  • Algoritmo
  • Pesos
Após configurações selecionadas o usuário poderá clicar em Treinar Modelo, para treinar o algoritmo e obter a Acurácia (Precisão do Modelo)

2 - Quais são os dados do Modelo padrão?

O usuario poderá utilizar um modelo padrão já previamente treinado pelo sistema, chamado no sistema de Modelo padrão, seus valores são:

  • Acurácia: 97,636 %
  • Proporção Teste / Treino: 25
  • Número de vizinhos : 5
  • Algoritmo: auto
  • Pesos: uniform
3 - O Modelo padrão é a melhor configuração do algoritmo?

Não, o modelo padrão não é a melhor configuração possível do algoritmo, o usuário poderá realizar a sua própria configuração com a finalidade de obter uma melhor acurácia e por consequência uma melhor previsão na sua recomendação de colheita, mas por não ser a melhor configuração possível o Modelo Padrão não é uma escolha ruim, ele é somente uma opção padrão para que não seja obrigatório, a configuração de um modelo por parte do usuário.

4 - O que a Proporção Teste / Treino afeta no modelo?

Os valores de Proporção Teste / Treino, são referentes a porcentagem de dados que serão utilizados para treino ou teste, quanto maior o valor, mais dados serão utilizados para teste e quanto menor, menos dados serão utilizados para teste, o valor pode ser configurado de 1 até 99.

5 - O que o Número de Vizinhos afeta no modelo?

O valor de Nuḿero de Vizinhos, define o número de vizinhos do algoritmo KNN que serão computados, quanto maior a quantidade de vizinhos mais dados próximos dos dados inseridos pelo usuário serão levados em conta na análise.

6 - O que o Algoritmo afeta no modelo?

O Algoritmo a ser escolhido dentre as opções auto, ball_tree, kd_tree e brute, são referentes ao algoritmo que será utilizado para computar os vizinhos próximos.

  • auto: Tentará decidir o algoritmo mais apropriado com base nos valores passados para o método de ajuste.
  • ball_tree : Utiliza o BallTree do Scikit learn para problemas generalizados de N pontos.
  • kd_tree : Utiliza o KDTree para problemas generalizados rápidos de N pontos.
  • brute : Utiliza a técnica de força bruta para achar os melhores pontos.

7 - O que os Pesos afetam no modelo?

Os dados de Pesos, são referentes aos pesos dos cálculos utilizados na distância entre os vizinhos.

  • uniform : Pesos uniformes, onde todos os pontos dos vizinhos tem um peso igual.
  • distance : Os pontos dos pesos tem o inverso de sua distância, onde os vizinhos mais próximos do ponto terão mais influência nos vizinhos mais distantes.

8 - Como funciona a recomendação de colheitas quando faço a inserção da minha análise de solo?

A recomendação da colheita se da a partir do modelo selecionado, juntamente com os dados de Nitrogênio, Fósforo, Potássio, Temperatura, umidade, pH e Chuva, inseridos pelo usuário, aṕos isso o Oraculus utiliza os dados da análise de solo e do modelo para determinar a melhor colheita a ser plantada.

10 - Como funciona a tela de Recomendar Colheeita?

Na tela de recomendar colheita o usuário poderá escolher uma colheita, e verificar dados estatísticos sobre os valores da colheita selecionada, podendo ajudar na tomada de alguma decisão caso o usuário já possua uma colheita. Nesta tela também possui a opção de se tirar um relatório PDF para uso posterior, caso necessário.

11 - Posso recuperar uma Análise ou Modelo deletado?

Não, após deletar uma Análise ou Modelo o mesmo não poderá mais ser recuperado, pois foi deletado do banco de dados.

12 - Como funciona o Dashboard do Modelo?

O Dashboard do Modelo, traz as informações do modelo além de trazer a Matriz de Confusão do modelo treinado. Com isso o usuário poderá ter informações visuais sobre o modelo.
